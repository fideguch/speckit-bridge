## FR-001: タスク作成

| 項目           | 内容                                                       |
| -------------- | ---------------------------------------------------------- |
| ID             | FR-001                                                     |
| 説明           | ユーザーがタイトル・説明・期限を指定してタスクを作成できる |
| 優先度         | Must                                                       |
| アクター       | 開発メンバー                                               |
| 事前条件       | ユーザーが認証済みであること                               |
| 事後条件       | タスクが作成され、Task エンティティとして保存される        |
| ビジネスルール | タイトルは1-200文字。期限は現在日時以降のみ                |
| 例外フロー     | タイトル未入力 → 400 エラー。認証切れ → 401 エラー         |

## FR-002: タスクステータス変更

| 項目           | 内容                                                           |
| -------------- | -------------------------------------------------------------- |
| ID             | FR-002                                                         |
| 説明           | タスクのステータスを Todo/InProgress/Done に変更できる         |
| 優先度         | Must                                                           |
| アクター       | 開発メンバー, 開発リーダー                                     |
| 事前条件       | タスクが存在すること                                           |
| 事後条件       | タスクのステータスが更新され、更新日時が記録される             |
| ビジネスルール | Done → Todo への逆戻りは不可。ステータス変更には監査ログを記録 |
| 例外フロー     | 存在しないタスクID → 404 エラー。不正な遷移 → 422 エラー       |

## FR-003: タスク割り当て

| 項目           | 内容                                                                     |
| -------------- | ------------------------------------------------------------------------ |
| ID             | FR-003                                                                   |
| 説明           | 開発リーダーがタスクを特定のユーザーに割り当てできる                     |
| 優先度         | Must                                                                     |
| アクター       | 開発リーダー                                                             |
| 事前条件       | タスクが存在し、割り当て先ユーザーがチームメンバーであること             |
| 事後条件       | タスクに assignee が設定される                                           |
| ビジネスルール | 1タスクにつき assignee は1人まで。リーダーのみ他人のタスクを再割り当て可 |
| 例外フロー     | 権限不足 → 403 エラー。存在しないユーザー → 404 エラー                   |

## FR-004: タスクコメント

| 項目           | 内容                                               |
| -------------- | -------------------------------------------------- |
| ID             | FR-004                                             |
| 説明           | タスクにコメントを追加できる                       |
| 優先度         | Should                                             |
| アクター       | 開発メンバー, 開発リーダー                         |
| 事前条件       | タスクが存在すること                               |
| 事後条件       | コメントが Comment エンティティとして保存される    |
| ビジネスルール | コメントは1-2000文字。コメント削除は投稿者本人のみ |
| 例外フロー     | タスクが存在しない → 404 エラー                    |
