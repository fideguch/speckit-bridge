# TaskFlow API

## 1. 基本情報

| 項目         | 内容                         |
| ------------ | ---------------------------- |
| 名称         | TaskFlow API                 |
| 一行説明     | チーム向けタスク管理REST API |
| 想定ユーザー | 5-50人規模の開発チーム       |

## 2. 背景と目的

### 背景

既存のタスク管理ツールはカスタマイズ性が低く、チーム固有のワークフローに対応できない。

### 成功の定義

- タスク作成から完了までの平均時間を30%短縮
- チームメンバーの週次タスク完了率80%以上
- API レスポンスタイム p95 < 200ms

## 3. ターゲットユーザー

| ペルソナ     | 説明                                   |
| ------------ | -------------------------------------- |
| 開発リーダー | チームのタスクを割り当て・進捗管理する |
| 開発メンバー | 自分のタスクを管理・ステータス更新する |

## 4. やりたいこと

### Must（必須）

- タスクの CRUD 操作
- ステータス管理（Todo/InProgress/Done）
- ユーザーへのタスク割り当て

### Should（推奨）

- タスクへのコメント機能
- 期限リマインダー通知

### Could（あれば良い）

- ダッシュボード集計 API

### Out of Scope

- フロントエンド UI
- モバイルアプリ
- 外部カレンダー連携

## 5. 制約・前提条件

- TypeScript + Node.js (Express)
- PostgreSQL 15+
- 開発期間: 4週間
- 初期ユーザー: 社内50名
- REST API のみ（GraphQL は将来検討）

## 6. 品質に関する期待値

- テストカバレッジ 80% 以上
- API ドキュメント自動生成（OpenAPI）
- CI/CD パイプライン必須
